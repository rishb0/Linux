# SSH Client — Connection Guide

---
## 1. Theory

### What Happens on the Client Side

- The SSH **client** initiates the connection to the server on port 22.
- On first connect to a new server, SSH asks you to verify and accept the server's **host key fingerprint** — saved to `~/.ssh/known_hosts`.
- On future connects, SSH compares the server's key against the saved one. If it doesn't match, it throws a warning.

### Client Authentication Methods

|Method|How It Works|Security|
|---|---|---|
|**Password**|User types their account password|Medium|
|**Public Key**|Client proves identity using private key matched to server's `authorized_keys`|High (recommended)|

## 4. Connecting to a Server — Step by Step

### Step 1 — Open a terminal

- **Linux / macOS:** Open Terminal.
- **Windows 10/11:** Open PowerShell, CMD, or Windows Terminal.
- **Windows 7 and below:** Open PuTTY (see Section 5).

### Step 2 — Connect using the SSH command

Basic syntax:

```bash
ssh username@server-ip
```

Example — connect as root:

```bash
ssh root@192.168.1.100
```

Example — connect as a regular user:

```bash
ssh risabh@192.168.1.29
```

### Step 3 — Accept the host key (first time only)

You'll see a prompt like:

```
The authenticity of host '192.168.1.100' can't be established.
ED25519 key fingerprint is SHA256:xxxxxxxxxxxxxxxxxx.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Type `yes` and press Enter. The server's key is saved to `~/.ssh/known_hosts`.

### Step 4 — Enter password

Type the account password and press Enter. Input is invisible — this is normal.

## 5. PuTTY — Windows 7 and Below

1. Download PuTTY from [https://www.putty.org](https://www.putty.org/)
2. Open PuTTY.
3. In **Host Name (or IP address)** field: enter the server IP e.g. `192.168.1.100`
4. **Port**: `22`
5. **Connection type**: select `SSH`
6. Click **Open**
7. Enter username and password when prompted.

> PuTTY also works on Windows 10/11 as an alternative to the built-in client.

## 6. Useful Connection Options

Connect on a non-default port:

```bash
ssh -p 2222 username@server-ip
```

Connect using a specific private key file:

```bash
ssh -i ~/.ssh/id_ed25519 username@server-ip
```

Run a single command on the remote server without opening an interactive shell:

```bash
ssh username@server-ip "command"
```

Example — check disk usage on remote server:

```bash
ssh root@192.168.1.100 "df -h"
```

Enable verbose output (useful for debugging):

```bash
ssh -v username@server-ip
```

More verbose:

```bash
ssh -vv username@server-ip
```

Maximum verbosity:

```bash
ssh -vvv username@server-ip
```

## 8. File Transfer Over SSH

### `scp` — Secure Copy

Copy a file from local machine to remote server:

```bash
scp /local/path/file.txt username@server-ip:/remote/path/
```

Copy a file from remote server to local machine:

```bash
scp username@server-ip:/remote/path/file.txt /local/path/
```

Copy an entire directory from local to remote:

```bash
scp -r /local/folder username@server-ip:/remote/path/
```

---

### `rsync` — Efficient Sync Over SSH

Sync local folder to remote:

```bash
rsync -avz /local/folder/ username@server-ip:/remote/folder/
```

Sync remote folder to local:

```bash
rsync -avz username@server-ip:/remote/folder/ /local/folder/
```

## 9. SSH Config File — Shortcuts for Frequent Connections

Create or edit the SSH client config file:

```bash
nano ~/.ssh/config
```

Add entries like this:

```
Host myserver
    HostName 192.168.1.100
    User root
    Port 22
    IdentityFile ~/.ssh/id_ed25519

Host devbox
    HostName 10.0.0.5
    User risabh
    Port 2222
```

Now connect using the alias instead of typing the full command:

```bash
ssh myserver
```

```bash
ssh devbox
```

## 10. SSH Key Management

Generate a new key pair (ed25519 — recommended):

```bash
ssh-keygen -t ed25519 -C "label_for_this_key"
```

Generate RSA key (older systems):

```bash
ssh-keygen -t rsa -b 4096
```

Copy your public key to the server (sets up key-based login):

```bash
ssh-copy-id username@server-ip
```

View your public key:

```bash
cat ~/.ssh/id_ed25519.pub
```

Add key to SSH agent (avoids re-entering passphrase every session):

```bash
ssh-add ~/.ssh/id_ed25519
```

List keys currently loaded in the SSH agent:

```bash
ssh-add -l
```

## 11. Common Errors and Fixes

### Error: `REMOTE HOST IDENTIFICATION HAS CHANGED`

```
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@ WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!        @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY!
```

**Cause:** Server's host key changed — OS reinstalled, VM redeployed with same IP, or IP reassigned to a different machine.

Remove the old key entry for that IP:

```bash
ssh-keygen -R 192.168.1.100
```

Then reconnect — SSH will ask you to accept the new key:

```bash
ssh root@192.168.1.100
```

Manual fix — open known_hosts and delete the line for that server:

```bash
nano ~/.ssh/known_hosts
```

Find and delete the line containing the server's IP or hostname, save, then reconnect.

---

### Error: `Connection refused`

**Cause:** SSH not running on server, wrong port, or firewall blocking port 22.

Check basic connectivity to the server:

```bash
ping server-ip
```

Try specifying the port explicitly:

```bash
ssh -p 22 username@server-ip
```

---

### Error: `Permission denied (publickey)`

**Cause:** Password auth is disabled on the server, wrong username, or wrong/missing key.

Use verbose mode to see what's failing:

```bash
ssh -v username@server-ip
```

Specify the key file explicitly:

```bash
ssh -i ~/.ssh/id_ed25519 username@server-ip
```

---

### Error: `ssh: connect to host x.x.x.x port 22: No route to host`

**Cause:** Network-level issue — client can't reach the server, or server firewall is dropping packets.

Test basic network reachability:

```bash
ping server-ip
```

---

### Error: `Could not resolve hostname`

**Cause:** Hostname doesn't resolve to an IP address.

Use the IP address directly instead:

```bash
ssh username@192.168.1.100
```
