# CentOS Stream 9 — Installation Interface Guide

---
## VM Requirements

|Component|Specification|
|---|---|
|**CPU**|2 cores|
|**RAM**|4 GB|
|**Storage**|50 GB|
|**Sub-Type**|Red Hat|
|**Version**|Red Hat x64 / Red Hat 9 (x64)|

---

## Boot Menu — Quick Decision Guide

|Situation|Use Option|
|---|---|
|Ready to install, media already verified|Option 1|
|Unsure about media integrity|Option 2|
|Hardware issues, recovery, or advanced troubleshooting|Option 3|

---

## Option 1 — Install CentOS Stream 9

- Boots directly into the **Anaconda Installer** (GUI).
- Guides through: language → partitioning → software selection → network config.
- **No pre-installation media check** is performed.
- Use when you're confident in your media and ready to install immediately.

---

## Option 2 — Test This Media & Install CentOS Stream 9

- Runs a **checksum validation (SHA-256)** on the installation media before proceeding.
- If the check passes, automatically continues into the installation process.
- If it fails, reports the corruption before any installation begins.
- Use when:
    - Using freshly written/newly created installation media.
    - Media might be damaged or corrupted.
    - ISO was downloaded from a non-official source.

---

## Option 3 — Troubleshooting

Advanced options menu for recovery and hardware-related issues.

### Sub-options

|Option|Purpose|When to Use|
|---|---|---|
|**Install using Text Mode**|Non-graphical (TUI) installation|Low-resource systems or incompatible GPU drivers|
|**Rescue a CentOS Stream System**|Boots into rescue mode to repair an existing install|Fixing bootloader, restoring files, system recovery|
|**Run a Memory Test**|Tests RAM using **Memtest86+**|Diagnosing hardware memory errors|
|**Boot From Local Drive**|Boots into the already-installed OS on the local disk|Exit installer; return to existing system|
|**Return to Main Menu**|Goes back to the initial boot menu|Re-select a different boot option|

- Use Option 3 when:
    - System cannot boot or install due to hardware/driver issues.
    - You need to recover a previously installed CentOS system.
    - You're diagnosing hardware (memory, graphics) or media problems.
