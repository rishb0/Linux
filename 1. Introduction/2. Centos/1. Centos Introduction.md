# CentOS 

---
## 1. Overview

- **CentOS** (Community ENTerprise Operating System) is a free, open-source Linux distro built from **RHEL source code**.
- Goal: enterprise-grade, RHEL binary-compatible environment at zero cost.
- Maintained by a community of contributors; no licensing fees.
- **EOL situation:** CentOS Linux 8 was discontinued in Dec 2021. Replaced by **CentOS Stream** (rolling). Users needing a stable RHEL clone moved to **AlmaLinux** or **Rocky Linux**.

---
## 2. Key Features

- **RHEL binary compatibility** — software compiled for RHEL runs on CentOS without modification.
- Enterprise tools included: **SELinux**, **systemd**, **Firewalld**.
- Free to use, modify, and redistribute (open-source).
- Regular security patches and long-term support (on traditional releases).

---

## 3. CentOS Versions

|Version|Based On|Status|Notes|
|---|---|---|---|
|**CentOS 7**|RHEL 7|EOL June 2024|Widely used; stable|
|**CentOS 8**|RHEL 8|EOL Dec 2021|Replaced by CentOS Stream|
|**CentOS Stream 9**|RHEL 9 upstream|Active|Rolling; preview of next RHEL minor release|

> **CentOS Stream** sits _between_ Fedora (upstream) and RHEL (downstream). It receives updates **before** RHEL — meaning it's a continuous preview of upcoming RHEL changes, not a downstream stable clone. Flow: **Fedora → CentOS Stream → RHEL**.

> ⚠️ There is no "CentOS 9" as a stable release. Only **CentOS Stream 9** exists, which is a rolling release — not equivalent to the old point-release CentOS model.

## 5. Security Features

### SELinux (Security-Enhanced Linux)

- A **Mandatory Access Control (MAC)** system built into the kernel.
- Goes beyond standard file permissions — defines policies on what processes can access which resources.
- Three modes:
    - `enforcing` — policies enforced; violations blocked and logged.
    - `permissive` — violations logged but not blocked (useful for debugging).
    - `disabled` — SELinux off entirely.
- Check status: `sestatus` | Change mode: `setenforce 0/1`

### Firewalld

- Default firewall management tool in CentOS 7+.
- Uses the concept of **zones** (e.g., `public`, `internal`, `trusted`) to define trust levels for network interfaces and traffic.
- Managed via `firewall-cmd`.

```bash
firewall-cmd --state                        # Check status
firewall-cmd --zone=public --list-all       # List rules in zone
firewall-cmd --permanent --add-port=80/tcp  # Allow port 80
firewall-cmd --reload                       # Apply permanent rules
```

## 6. System Management (systemd)

CentOS uses **systemd** as its init system and service manager.

```bash
systemctl start <service>     # Start a service
systemctl stop <service>      # Stop a service
systemctl restart <service>   # Restart a service
systemctl enable <service>    # Enable at boot
systemctl disable <service>   # Disable at boot
systemctl status <service>    # Check status

journalctl -u <service>       # View logs for a specific service
journalctl -xe                # View recent system logs with details
```

## 7. Network Management

- **NetworkManager** — manages wired/wireless connections; runs as a daemon.
- **nmcli** — command-line tool to configure NetworkManager.
- **Firewalld + firewall-cmd** — for firewall/traffic filtering (covered above).

```bash
nmcli device status           # Show all network devices
nmcli connection show         # List connections
nmcli connection up <name>    # Activate a connection
```

## 10. Performance Tuning

- **Minimal install** — installs only essential packages; reduces attack surface and resource usage.
- **`tuned`** — daemon that applies tuning profiles (e.g., `throughput-performance`, `latency-performance`) based on workload.
- **`sysctl`** — adjusts kernel parameters at runtime (e.g., network buffers, file descriptors).

```bash
tuned-adm list                        # List available profiles
tuned-adm active                      # Show active profile
tuned-adm profile throughput-performance  # Apply a profile
sysctl -a                             # List all kernel parameters
sysctl -w net.ipv4.ip_forward=1       # Set a parameter at runtime
```

## 11. System Administration Tasks

### User Management

```bash
useradd <user>          # Create user
passwd <user>           # Set password
groupadd <group>        # Create group
usermod -aG <group> <user>  # Add user to group
```

### File System Management

- Supported file systems: **ext4**, **XFS** (default on RHEL/CentOS), **Btrfs**.
- Tools: `fdisk`, `parted` (partitioning) | `mount` / `umount` | `df -h`, `du -sh`.

### Cron Jobs

```bash
crontab -e      # Edit current user's cron jobs
crontab -l      # List cron jobs
# Format: MIN HOUR DOM MON DOW COMMAND
# Example: 0 2 * * * /usr/bin/rsync -av /src /backup
```

### Backup Tools

|Tool|Type|
|---|---|
|`rsync`|Incremental file/dir sync and backup|
|`tar`|Archive creation and extraction|
|**Bacula** / **Amanda**|Enterprise-grade backup solutions|
