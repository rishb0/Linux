# CentOS ISO — Types, Checksums & Verification

---
## Quick Reference

| ISO Type       | Size           | Purpose                                                             |
| -------------- | -------------- | ------------------------------------------------------------------- |
| DVD            | ~4.5 GB        | Full install with GUI, server, and desktop options                  |
| Minimal        | ~800 MB – 1 GB | Core packages only; no GUI; server installs                         |
| Everything     | ~6–7 GB+       | All available packages; offline/air-gapped installs                 |
| NetInstall     | ~500 MB        | Boots a small image; fetches packages over network                  |
| LiveCD/LiveDVD | ~1.5–2 GB      | Live bootable environment to test CentOS without installing         |
| Boot           | ~300–500 MB    | Minimal bootable image to start installation; pairs with NetInstall |

|Checksum Type|Security Level|Status|
|---|---|---|
|MD5|Low|Legacy; basic integrity only|
|SHA-1|Medium|Being phased out|
|SHA-256|High|Most commonly used; recommended|
|SHA-512|Highest|More secure than SHA-256|

## 1. CentOS ISO Types

### 1.1 DVD ISO _(Full Installation)_

- Complete installation with GUI, server, and desktop options.
- Size: **~4.5 GB**
- Contains full CentOS package set — all major tools and libraries.
- Example: `CentOS-8-x86_64-dvd1.iso`

### 1.2 Minimal ISO

- Core set of packages for a minimal **server installation** (no GUI).
- Size: **~800 MB – 1 GB**
- Only essential system files; command-line based.
- Example: `CentOS-8-x86_64-minimal.iso`

### 1.3 Everything ISO

- Contains **all packages** available for CentOS.
- Size: **~6–7 GB or more**
- Ideal for **offline installations** or **air-gapped networks** (no internet access).
- Example: `CentOS-8-x86_64-everything.iso`

### 1.4 Network Install (NetInstall) ISO

- Small bootable image that fetches packages **from the network** during installation.
- Size: **~500 MB**
- Requires a working internet connection or local mirror.
- Example: `CentOS-8-x86_64-netinstall.iso`

### 1.5 LiveCD / LiveDVD ISO

- Fully bootable environment with GUI and basic tools.
- Size: **~1.5–2 GB**
- Used to **test CentOS without installing** it on the machine.
- Example: `CentOS-8-x86_64-live.iso`

### 1.6 Boot ISO

- Minimal bootable image to **start the installation process only**.
- Size: **~300–500 MB**
- Does not contain packages — used in **network installation setups**, typically paired with NetInstall.
- Example: `CentOS-8-x86_64-boot.iso`

## 2. Checksum Files

Checksums verify that the downloaded ISO has not been corrupted or tampered with. CentOS provides checksum files alongside every ISO.

### 2.1 MD5

- Oldest checksum method; used for basic integrity checks.
- **Not recommended for security verification** — MD5 is cryptographically broken.
- Example file: `CentOS-8-x86_64-dvd1.iso.md5`
- Tool: `md5sum`

### 2.2 SHA-1

- More secure than MD5 but **being phased out** in favour of SHA-256.
- Example file: `CentOS-8-x86_64-dvd1.iso.sha1`
- Tool: `sha1sum`

### 2.3 SHA-256 _(Recommended)_

- Most commonly used; current standard for file integrity verification.
- Example file: `CentOS-8-x86_64-dvd1.iso.sha256`
- Tool: `sha256sum`

### 2.4 SHA-512

- Stronger than SHA-256; used when highest security assurance is needed.
- Example file: `CentOS-8-x86_64-dvd1.iso.sha512`
- Tool: `sha512sum`

## 3. Manifest Files

- **Purpose:** Lists all RPM packages included in the ISO image.
- **Use cases:**
    - Verify the package contents of an ISO before installation.
    - Audit what is bundled in a given release.
    - Useful for planning offline/air-gapped deployments.
- Example file: `CentOS-8-x86_64-dvd1.iso.manifest`

## 5. How to Verify an ISO

After downloading the ISO and its corresponding checksum file:

**Step 1 — Calculate the hash of your downloaded ISO:**

```bash
sha256sum CentOS-8-x86_64-dvd1.iso
```

**Step 2 — Compare with the provided checksum:**

```bash
# If the checksum file is downloaded alongside the ISO:
sha256sum -c CentOS-8-x86_64-dvd1.iso.sha256
```

Output should say: `CentOS-8-x86_64-dvd1.iso: OK`

> If the hashes do **not** match, the file is corrupted or tampered — delete it and re-download.

**Other checksum tools:**

```bash
md5sum    CentOS-8-x86_64-dvd1.iso
sha1sum   CentOS-8-x86_64-dvd1.iso
sha512sum CentOS-8-x86_64-dvd1.iso
```

## 6. When to Use Which ISO

|Scenario|Recommended ISO|
|---|---|
|Full desktop/server install with GUI|DVD ISO|
|Minimal headless server install|Minimal ISO|
|Offline or air-gapped environment|Everything ISO|
|Install over network / internet|NetInstall ISO|
|Test CentOS before committing|LiveCD/LiveDVD ISO|
|Network install bootstrap only|Boot ISO|

---

## 7. Key Considerations

- **Choose the right ISO** based on your installation type — minimal for servers, DVD for full setups, Everything for offline.
- **Bandwidth matters** — use Minimal or NetInstall on slow connections; DVD/Everything for fast local networks.
- **Always verify** downloaded ISOs with checksums before installing — prevents corrupted or tampered installs.
- **Air-gapped environments** — use Everything ISO since no internet access is available during installation.
- **Network installs** — Boot ISO and NetInstall ISO both require a reachable package mirror (internet or local).
