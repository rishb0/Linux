# Linux Distributions — Complete Notes

---

## What is a Linux Distribution?

A **Linux distribution (distro)** is a complete operating system built on top of the **Linux kernel**, packaged with:

- **GNU userspace** (core tools: bash, coreutils, glibc, etc.)
- A **package manager** (handles software installation, updates, and dependency resolution)
- An **init system** (e.g., systemd, OpenRC, runit)
- Optional: desktop environment, display server, and pre-installed applications

Distros form **family trees** — most are derived from a parent distro and inherit its package format, package manager, and core philosophy. Understanding the family instantly tells you how the system works.

---
## The Linux Kernel

- Developed by **Linus Torvalds** in **1991**.
- The **core** of every Linux distribution.
- Acts as the **interface between hardware and software**.
- Responsibilities:
    - **Process management** — scheduling, creation, termination of processes
    - **Memory management** — RAM allocation, virtual memory, paging
    - **Device drivers** — communication with hardware (disks, network cards, GPUs)
    - **File system management** — reading/writing data via file systems (ext4, XFS, etc.)
    - **System calls** — the API through which user-space programs interact with the kernel
    - **Networking** — TCP/IP stack, socket management
    - **Security** — permissions, namespaces, cgroups, SELinux/AppArmor enforcement

## 1. General-Purpose Distributions

Versatile distros that can be used on desktops, laptops, and servers.

---
### 1.1 Debian-based
- **Package manager:** `apt` / `dpkg`
- **Package format:** `.deb`
- Known for stability, large repositories, and being the upstream for the most popular desktop distros.

| Distro              | Notes                                                                               |
| ------------------- | ----------------------------------------------------------------------------------- |
| **Debian**          | Parent distro; extremely stable; basis for most entries below                       |
| **Ubuntu**          | Most popular desktop/server distro; backed by Canonical; various flavours available |
| **Linux Mint**      | Beginner-friendly; Cinnamon, XFCE, or MATE edition; Windows-like UI                 |
| **Elementary OS**   | macOS-like aesthetic; design and simplicity focused                                 |
| **Zorin OS**        | For Windows migrants; polished, familiar interface                                  |
| **ParrotOS**        | Security, privacy, and development-oriented                                         |
| **Kali Linux**      | Penetration testing and security auditing; 600+ tools preinstalled                  |
| **Raspberry Pi OS** | Formerly Raspbian; optimised for Raspberry Pi ARM hardware                          |
| **MX Linux**        | Lightweight and fast; built on Debian Stable; community favourite                   |
| **Deepin**          | Chinese-developed; beautiful Deepin Desktop Environment (DDE)                       |
| **SparkyLinux**     | Lightweight; rolling and semi-rolling releases available                            |
| **Lubuntu**         | Official Ubuntu flavour; LXQt desktop; very lightweight                             |
| **Xubuntu**         | Official Ubuntu flavour; XFCE desktop                                               |
| **Kubuntu**         | Official Ubuntu flavour; KDE Plasma desktop                                         |
| **Ubuntu MATE**     | Official Ubuntu flavour; MATE desktop; classic layout                               |
| **Ubuntu Studio**   | Official Ubuntu flavour; low-latency kernel; audio/video/graphics production        |

> **Note:** Lubuntu, Xubuntu, Kubuntu, Ubuntu MATE, and Ubuntu Studio are **official Ubuntu flavours** — not independent distros. They share the same Debian/Ubuntu base and APT repositories, differing only in default desktop environment.

---
### 1.2 Red Hat-based

- **Package manager:** `dnf` (modern) / `yum` (legacy, RHEL 7 and older)
- **Package format:** `.rpm`
- Dominant in enterprise and server environments.

| Distro                              | Notes                                                                               |
| ----------------------------------- | ----------------------------------------------------------------------------------- |
| **Red Hat Enterprise Linux (RHEL)** | Commercial enterprise OS; industry standard for production servers and data centres |
| **Fedora**                          | Community-driven; cutting-edge features; upstream testing ground for RHEL           |
| **CentOS Stream**                   | Rolling preview of upcoming RHEL; replaced traditional CentOS Linux                 |
| **AlmaLinux**                       | Free, binary-compatible RHEL rebuild; recommended CentOS Linux replacement          |
| **Rocky Linux**                     | Free RHEL rebuild; founded by CentOS co-founder Gregory Kurtzer                     |
| **Oracle Linux**                    | RHEL-compatible; includes Oracle's Unbreakable Enterprise Kernel (UEK)              |
| **Scientific Linux**                | _(Discontinued 2021)_ Was used by CERN and Fermilab; succeeded by AlmaLinux/Rocky   |

> **CentOS Linux** (traditional, point-release) reached **End of Life on December 31, 2021**. It has been replaced by **CentOS Stream** (a rolling upstream preview of RHEL, not a downstream rebuild). Users needing a free, stable RHEL clone should use **AlmaLinux** or **Rocky Linux**.

---
### 1.3 Arch-based

- **Package manager:** `pacman`
- **Package format:** `.pkg.tar.zst`
- **Release model:** Rolling — no fixed versions; system is continuously updated.
- Community packages via the **AUR (Arch User Repository)**.
- The **Arch Wiki** is widely regarded as the best Linux documentation resource.
- Follows the **KISS principle** (Keep It Simple, Stupid) — minimal defaults, full user control.

| Distro          | Notes                                                                                      |
| --------------- | ------------------------------------------------------------------------------------------ |
| **Arch Linux**  | Parent distro; fully manual setup; maximum control and minimalism                          |
| **Manjaro**     | Beginner-friendly Arch; hardware detection; slightly delayed rolling updates for stability |
| **EndeavourOS** | Near-vanilla Arch with a guided installer; active community                                |
| **ArcoLinux**   | Educational Arch-based; teaches Linux step by step                                         |
| **Antergos**    | _(Discontinued 2019)_ Was a graphical Arch installer                                       |
| **Apricity OS** | _(Discontinued 2017)_ Arch-based; no longer maintained                                     |

---
### 1.4 SUSE-based  (**Software und System-Entwicklung**)

- **Package manager:** `zypper`
- **Package format:** `.rpm`
- Known for **YaST (Yet another Setup Tool)** — a powerful graphical and text-based system configuration utility.

| Distro                                  | Notes                                                           |
| --------------------------------------- | --------------------------------------------------------------- |
| **openSUSE Leap**                       | Stable, point-release; community distro aligned with SLES       |
| **openSUSE Tumbleweed**                 | Rolling release variant; always latest stable packages          |
| **SUSE Linux Enterprise Server (SLES)** | Commercial enterprise OS; heavily used in SAP HANA environments |

---
### 1.5 Gentoo-based

- **Package manager:** `emerge` (Portage)
- **Package format:** Source tarballs — all software is **compiled from source** on the user's machine.
- Allows deep optimisation for specific hardware (CPU flags, USE flags).
- Requires significantly more time and expertise than binary distros.

| Distro              | Notes                                                                      |
| ------------------- | -------------------------------------------------------------------------- |
| **Gentoo**          | Parent distro; maximum customisation; source-compiled packages             |
| **Funtoo**          | Fork by Gentoo's original creator Daniel Robbins; community-driven         |
| **Calculate Linux** | Russian-developed; desktop and server focus; binary packages available     |
| **Chrome OS**       | Google's Chromebook OS; Gentoo-based build system; Google custom userspace |
| **Sabayon Linux**   | _(Discontinued 2021)_ Community moved to Macaroni OS                       |

---
### 1.6 Slackware-based

- **Package manager:** `pkgtool` / `slackpkg` — **no automatic dependency resolution**
- **Package format:** `.txz` (tar + xz)
- The **oldest surviving Linux distribution** (1993).
- Strictly Unix-like; minimalist philosophy; close to Unix roots.

| Distro          | Notes                                                                |
| --------------- | -------------------------------------------------------------------- |
| **Slackware**   | Parent distro; oldest active Linux distro; minimalist, Unix-orthodox |
| **Zenwalk**     | Lightweight; one-application-per-task philosophy                     |
| **VectorLinux** | Lightweight; designed for older hardware                             |
| **Salix OS**    | User-friendly Slackware derivative                                   |

---
### 1.7 Independent Distributions

Built from scratch — not derived from any major family above.

| Distro                       | Notes                                                                                                      |
| ---------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **NixOS**                    | Declarative, reproducible system configuration; uses the Nix package manager                               |
| **Solus**                    | Independent desktop-first distro; uses `eopkg`; focused on simplicity and performance                      |
| **Clear Linux**              | Intel-developed; performance-optimised for Intel hardware and cloud workloads                              |
| **Alpine Linux**             | Extremely minimal and security-hardened; uses musl libc + BusyBox; default Docker base image               |
| **Puppy Linux**              | Ultra-lightweight; runs entirely from RAM; excellent for rescue and older hardware                         |
| **Tails**                    | Privacy-first live OS; all traffic routed through Tor; leaves no trace on host                             |
| **Qubes OS**                 | Security via compartmentalisation; each app runs in an isolated Xen VM                                     |
| **Tiny Core Linux**          | Possibly the world's smallest functional Linux (~11 MB); boots into RAM                                    |
| **LFS (Linux From Scratch)** | A guided book/project to build a complete Linux system from source; educational, not a downloadable distro |
| **CRUX**                     | Minimalist; ports-based package system; for experienced users                                              |
| **Source Mage**              | Source-based; spell-based package system called Sorcery                                                    |

## 7. Non-GNU Linux Distributions

These use the **Linux kernel** but replace or significantly reduce the **GNU userspace** — the collection of core utilities developed by the GNU Project (bash, glibc, coreutils, etc.). Because of this, they are technically "Linux" but not "GNU/Linux."

| OS            | Notes                                                                        |
| ------------- | ---------------------------------------------------------------------------- |
| **Android**   | Linux kernel + Bionic libc + ART (Android Runtime); no GNU tools by default  |
| **Chrome OS** | Linux kernel + Gentoo-based build system + Google's custom userspace         |
| **Tizen OS**  | Linux Foundation + Samsung; used in smart TVs, smartwatches, and IoT devices |
