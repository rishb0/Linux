## 1. What is an IP Address?

An IP address, which stands for **Internet Protocol address**, is a unique numerical label that is assigned to every device connected to a network that uses the Internet Protocol for communication. Think of it exactly like a postal address for your house — without it, no one knows where to send data, and your device does not know where to send its replies either.

Every piece of data that travels across a network is broken into small units called **packets**. Every single packet carries two critical pieces of information: a **source IP address** (the address of the device that sent it) and a **destination IP address** (the address of the device it is going to). Routers and switches along the way read these addresses and decide how to forward the packet toward its destination.

Without IP addresses, the entire internet — and every local network inside every office, home, and data center — would simply not function. IP addresses are the addressing system that makes all network communication possible.

## 2. Why IP Addresses Matter in Ethical Hacking

In ethical hacking and penetration testing, the IP address is almost always your starting point for everything. Understanding IP addresses deeply is not optional — it is foundational to every single thing you will do in this field.

**Reconnaissance:** When you perform reconnaissance on a target organization, you are finding out what IP addresses they own, what IP ranges belong to them, what services are running on those IPs, and how those IPs are organized across their network infrastructure. This is the first phase of any penetration test.

**Scoping:** When a client gives you a penetration test engagement, the scope — meaning the exact set of systems you are authorized to test — is almost always defined by a list of IP addresses or IP address ranges. Going outside that scope by touching an IP not in the list can have serious legal consequences.

**Log Analysis and Forensics:** When you analyze attack traffic, server logs, firewall logs, or malware behavior, IP addresses tell you who is communicating with whom, from where, and to what. Tracing an attacker's path through a network is largely an exercise in following IP addresses.

**Network Mapping:** Before you can attack anything, you need to know what is there. Network mapping (using tools like Nmap) is entirely based on sending probes to IP addresses and interpreting the responses.

In short, everything in networking revolves around IP addresses, and therefore everything in hacking and security revolves around IP addresses too.

## 3. IPv4 — Deep Dive

### What IPv4 Is

IPv4 is the fourth version of the Internet Protocol and is still the most widely used version across the internet today, even though it has existed since the early 1980s. An IPv4 address is a **32-bit number**, which means it is made up of 32 binary digits (0s and 1s).

### How IPv4 is Written

Rather than writing all 32 bits out, IPv4 addresses are written in a human-readable format called **dotted-decimal notation**. The 32 bits are divided into four groups of 8 bits each, and each group is converted from binary into a decimal number between 0 and 255. These four decimal numbers are then separated by dots.

```
Example: 192.168.1.100
```

Each of these four groups is called an **octet** because it contains exactly 8 bits. The four octets together give you the full 32-bit address.

### How Many IPv4 Addresses Exist

Because the address is 32 bits long, the total number of possible IPv4 addresses is 2³² = **4,294,967,296**, which is approximately 4.3 billion addresses. This seems like a lot, but it turned out to be severely insufficient for the modern internet, which has billions of connected devices. This exhaustion of IPv4 addresses is one of the main reasons technologies like NAT (Network Address Translation) were created, and it is also the reason IPv6 was developed.

### Range of Each Octet

Each octet can hold a value from **0 to 255** because 8 bits can represent values from 00000000 (= 0) to 11111111 (= 255). This gives 256 possible values per octet (0 through 255 inclusive).

```
Minimum IPv4 address:  0.0.0.0
Maximum IPv4 address:  255.255.255.255
```

## 4. Binary Representation — The Foundation You Must Understand

Every IP address is, at its core, a binary number. If you do not understand binary, you will not be able to do subnetting, understand netmasks, or understand how routers make forwarding decisions. This section teaches you binary as it applies to IP addresses.

### What Binary Is

Binary is the base-2 number system, meaning it uses only two digits: 0 and 1. Every position in a binary number represents a power of 2, starting from the rightmost position which represents 2⁰ = 1, then 2¹ = 2, then 2² = 4, and so on.

For an 8-bit octet, the positional values from left to right are:

```
Position:  8th   7th   6th   5th   4th   3rd   2nd   1st
Value:     128    64    32    16     8     4     2     1
```

### Converting Decimal to Binary

To convert a decimal number to binary, you check each positional value from largest (128) to smallest (1) and ask: "Does this value fit into what I have left?"

**Example — Converting 192 to binary:**

- 192 ÷ 128 = fits (192 − 128 = 64 remaining) → write **1**
- 64 ÷ 64 = fits (64 − 64 = 0 remaining) → write **1**
- 0 ÷ 32 = does not fit → write **0**
- 0 ÷ 16 = does not fit → write **0**
- 0 ÷ 8 = does not fit → write **0**
- 0 ÷ 4 = does not fit → write **0**
- 0 ÷ 2 = does not fit → write **0**
- 0 ÷ 1 = does not fit → write **0**

Result: `192 = 11000000`

**Example — Converting 168 to binary:**

- 168 − 128 = 40 → **1**
- 40 − 64 = does not fit → **0**
- 40 − 32 = 8 → **1**
- 8 − 16 = does not fit → **0**
- 8 − 8 = 0 → **1**
- 0 − 4 = does not fit → **0**
- 0 − 2 = does not fit → **0**
- 0 − 1 = does not fit → **0**

Result: `168 = 10101000`

### Converting Binary to Decimal

To go from binary back to decimal, you simply add up the positional values wherever there is a 1.

**Example — Converting 11000000 to decimal:**

```
1×128 + 1×64 + 0×32 + 0×16 + 0×8 + 0×4 + 0×2 + 0×1
= 128 + 64
= 192
```

### Full IP Address in Binary

Here is the IP address `192.168.1.100` fully written in binary:

```
192  =  11000000
168  =  10101000
  1  =  00000001
100  =  01100100

Full:   11000000.10101000.00000001.01100100
```

### Binary Conversion Quick Reference Table

|Decimal|Binary|
|---|---|
|0|00000000|
|1|00000001|
|128|10000000|
|192|11000000|
|224|11100000|
|240|11110000|
|248|11111000|
|252|11111100|
|254|11111110|
|255|11111111|

These specific values appear constantly in subnet masks, so memorizing them will save you a lot of time.

## 5. IPv6 — Deep Dive

### Why IPv6 Was Created

As explained above, IPv4 only provides approximately 4.3 billion addresses, and with the explosion of internet-connected devices, this pool was exhausted. IPv6 was created to solve the address exhaustion problem permanently.

### What IPv6 Is

IPv6 is a **128-bit** address, which means it is four times longer than IPv4. The total number of possible IPv6 addresses is 2¹²⁸ = approximately **3.4 × 10³⁸**, which is a number so astronomically large that we will never run out. To give you a sense of scale: this is enough to give every atom on the surface of the Earth its own IP address many times over.

### How IPv6 is Written

IPv6 addresses are written as eight groups of four **hexadecimal** digits, separated by colons. Hexadecimal (hex) is base-16, using digits 0–9 and letters a–f, where a=10, b=11, c=12, d=13, e=14, f=15.

```
Full example:   2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

### IPv6 Shortening Rules

IPv6 addresses can be shortened using two rules, which you will frequently encounter:

**Rule 1 — Leading zeros within any group can be removed:**

```
0db8  →  db8
0000  →  0
```

**Rule 2 — One or more consecutive groups of all zeros can be replaced with `::` (but only once in an address):**

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
→  2001:db8:85a3::8a2e:370:7334
```

### IPv6 Features Relevant to Ethical Hacking

IPv6 has several important features that you need to be aware of as an ethical hacker:

**No NAT required:** Because addresses are so plentiful, every device can have its own globally routable public IPv6 address. This means the "hiding" effect that NAT provided in IPv4 networks does not exist by default in IPv6.

**Built-in IPSec support:** IPv6 was designed with IPSec (the protocol suite for encrypting and authenticating IP traffic) as a mandatory component, though in practice it is not always enforced.

**SLAAC (Stateless Address Autoconfiguration):** Devices can automatically configure their own IPv6 addresses without a DHCP server by combining a network prefix advertised by the router with a host identifier derived from the device's MAC address. This means scanning IPv6 networks is much harder because you cannot simply enumerate a sequential range.

**Dual-stack environments:** Most modern networks run both IPv4 and IPv6 simultaneously. As an ethical hacker, you must always scan for both protocols because a service might be firewalled on IPv4 but exposed on IPv6, or vice versa. Missing IPv6 during reconnaissance is a real and common mistake.

### IPv6 Special Addresses

|Address|Purpose|
|---|---|
|`::1`|Loopback address (equivalent to 127.0.0.1 in IPv4)|
|`fe80::/10`|Link-local addresses — auto-assigned, only valid on the local link|
|`fc00::/7`|Unique local addresses — equivalent to IPv4 private addresses|
|`ff00::/8`|Multicast addresses|

## 6. Classes of IP Addresses (Classful Addressing)

Before modern subnetting techniques took over, IP addresses were divided into fixed categories called **classes**. This system is called **classful addressing**. Although modern networks use classless addressing (CIDR) instead, you will still encounter these class names constantly — in documentation, exam questions, and conversations with network engineers. You must know them.

### Class A

Class A addresses are designed for very large networks. The **first octet** identifies the network, and the **remaining three octets** identify hosts within that network. The first octet of a Class A address ranges from **1 to 126**. The first bit of a Class A address is always 0 (binary), which is how a router classifies it.

```
Range:          1.0.0.0  to  126.255.255.255
Network bits:   8  (first octet)
Host bits:      24 (last three octets)
Max hosts:      2²⁴ − 2  =  16,777,214 hosts per network
Example:        10.0.0.1
```

Class A was originally assigned to very large organizations, governments, and internet backbone providers. Today many of these blocks are subdivided.

### Class B

Class B addresses are for medium-to-large networks. The **first two octets** identify the network, and the **last two octets** identify hosts. The first octet ranges from **128 to 191**, and the first two bits are always 10 in binary.

```
Range:          128.0.0.0  to  191.255.255.255
Network bits:   16 (first two octets)
Host bits:      16 (last two octets)
Max hosts:      2¹⁶ − 2  =  65,534 hosts per network
Example:        172.16.0.1
```

### Class C

Class C addresses are for small networks. The **first three octets** identify the network, and only the **last octet** identifies hosts. The first octet ranges from **192 to 223**, and the first three bits are always 110 in binary.

```
Range:          192.0.0.0  to  223.255.255.255
Network bits:   24 (first three octets)
Host bits:      8  (last octet)
Max hosts:      2⁸ − 2  =  254 hosts per network
Example:        192.168.1.1
```

Most home networks, small offices, and lab environments fall into Class C. The subnet mask for a default Class C network is `255.255.255.0` or `/24`.

### Class D

Class D addresses are **not used for regular host addressing**. They are reserved for **multicast**, which is a method of sending a single packet to a group of devices simultaneously (as opposed to broadcast, which goes to all devices, or unicast, which goes to one device). The first octet ranges from **224 to 239**.

```
Range:  224.0.0.0  to  239.255.255.255
Use:    Multicast groups (e.g., routing protocols, streaming media)
```

### Class E

Class E addresses are reserved for **experimental and research use** and are never assigned to regular devices. The first octet ranges from **240 to 255**.

```
Range:  240.0.0.0  to  255.255.255.255
Use:    Reserved / experimental
```

### Why 127 is Skipped in Class A

You may notice that Class A goes from 1 to 126 and not to 127. The address range **127.0.0.0 to 127.255.255.255** is entirely reserved for **loopback** purposes (explained in Section 8), and therefore it is excluded from Class A's usable range.

### Visual Summary of Classes

```
First Octet Range    Class    Default Mask     Purpose
1   – 126            A        /8               Very large networks
128 – 191            B        /16              Medium networks
192 – 223            C        /24              Small networks
224 – 239            D        N/A              Multicast
240 – 255            E        N/A              Experimental
```

## 7. Public vs. Private IP Addresses

This distinction is one of the most critical concepts for anyone working in networking or cybersecurity.

### Public IP Addresses

A **public IP address** is a globally routable address on the internet. It is unique across the entire internet — no two devices on the public internet can have the same public IP address at the same time. Public IPs are assigned and managed by **IANA** (Internet Assigned Numbers Authority) and distributed through **Regional Internet Registries** (RIRs) such as ARIN (North America), RIPE (Europe), and APNIC (Asia-Pacific). Your ISP (Internet Service Provider) then assigns public IPs to its customers.

When you visit a website, the web server sees your public IP address as the source of your request. When you are doing OSINT (Open Source Intelligence) or external reconnaissance against a target organization, the IP addresses you find registered to that organization will be their public IPs.

### Private IP Addresses

**Private IP addresses** are ranges that have been specifically reserved for use inside local networks (LANs). They are defined by **RFC 1918**. Private IPs are **not routable on the public internet** — a router on the internet will drop a packet addressed to a private IP because it has no route for it.

The three private ranges are:

```
10.0.0.0    –  10.255.255.255      (Class A private, /8)
                 — supports ~16.7 million addresses
                 — often used in large corporate networks

172.16.0.0  –  172.31.255.255      (Class B private, /12)
                 — supports ~1 million addresses
                 — often used in medium-sized networks

192.168.0.0 –  192.168.255.255     (Class C private, /16)
                 — supports ~65,000 addresses
                 — used in home and small office networks
```

Because private IPs are not globally unique, many different organizations can all use `192.168.1.1` internally without any conflict, because that traffic never leaves their private network. The translation between private IPs and the public IP is handled by **NAT** (covered in Section 12).

### Ethical Hacking Context

When you are doing an **external penetration test** (testing from the internet, as an outsider would attack), you are working with public IP addresses. When you are doing an **internal penetration test** (testing from inside the organization's network, as an insider or a post-breach attacker would operate), you will almost always be working with private IP addresses in the RFC 1918 ranges. This difference in perspective is enormous — internal testing reveals far more attack surface than external testing.

## 8. Special and Reserved IP Addresses

### Loopback Address — 127.0.0.1

The loopback address **always refers to the local machine itself**. When a device sends any traffic to `127.0.0.1`, that traffic never actually goes out onto the network. The operating system's networking stack intercepts it and loops it back to the same machine. It is used to test whether the local machine's networking software is working correctly, and applications often use it to communicate with other services running on the same machine.

The hostname **`localhost`** resolves to `127.0.0.1` by default on virtually every operating system. The entire **`127.0.0.0/8`** range is reserved for loopback, meaning any address from `127.0.0.1` to `127.255.255.255` loops back to localhost, but `127.0.0.1` is the one universally used.

In IPv6, the loopback address is **`::1`**.

### Default Gateway

The **default gateway** is the IP address of the router on your local network. When your device wants to send traffic to any IP address that is not on its own local subnet, it sends that traffic to the default gateway, and the router then forwards it onward toward the destination. In most home and small office networks, the default gateway is something like `192.168.1.1` or `192.168.0.1`.

Identifying the default gateway is always one of the first things you do when you gain access to a machine during a penetration test, because it tells you the address of the router and helps you map the network topology.

### Network Address

The **network address** is the **first** IP address in any subnet. It is used to identify the subnet itself and cannot be assigned to any device. For example, in the network `192.168.1.0/24`, the network address is `192.168.1.0`.

### Broadcast Address

The **broadcast address** is the **last** IP address in any subnet. Traffic sent to the broadcast address is delivered to all devices on that subnet simultaneously. It also cannot be assigned to any device. For example, in the network `192.168.1.0/24`, the broadcast address is `192.168.1.255`. This is why a `/24` subnet has 254 usable host addresses rather than 256 — the first and last are reserved.

### APIPA — 169.254.0.0/16

**APIPA** stands for **Automatic Private IP Addressing**. When a device is configured to obtain its IP address automatically via DHCP but cannot reach a DHCP server, Windows (and some other operating systems) will automatically assign itself an address in the range **169.254.0.0 to 169.254.255.255**. This allows devices on the same local network to still communicate with each other even without a DHCP server, but they will have no internet access.

If you see a device with an IP address in the `169.254.x.x` range during a penetration test or network scan, it means that device failed to obtain a proper IP address from DHCP and almost certainly has no outbound network connectivity. This is a useful diagnostic indicator.

### 0.0.0.0

The address **0.0.0.0** is used in routing tables and configurations to represent "any address" or "all addresses." For example, a default route in a routing table is written as `0.0.0.0/0`, meaning "for any destination not matched by a more specific route, use this path." On a host, an application listening on `0.0.0.0` means it is listening on all available network interfaces.

### 255.255.255.255

This is the **limited broadcast address**. Traffic sent to `255.255.255.255` is broadcast to all hosts on the immediate local network, but routers do not forward it — it stays on the local segment.

## 9. Subnet Mask — What it is and Why it Exists

### The Problem Subnet Masks Solve

Every device on a network needs to know two things when it wants to send a packet: "Is the destination on my own local network, or is it on a remote network?" If the destination is local, the device can send the packet directly. If the destination is remote, the device sends the packet to its default gateway. The device figures out which case applies using the **subnet mask**.

### What a Subnet Mask Is

A **subnet mask** is a 32-bit number (written in the same dotted-decimal format as an IPv4 address) that defines which portion of an IP address is the **network portion** and which portion is the **host portion**.

The rule is simple: in the subnet mask, all **1-bits** mark the network portion of the IP address, and all **0-bits** mark the host portion. In a valid subnet mask, all the 1s are always contiguous (grouped together on the left), and all the 0s come after them on the right. You will never see a subnet mask like `11001111` where the 1s and 0s are interleaved.

### Example

```
IP Address:    192.168.1.100
               11000000.10101000.00000001.01100100

Subnet Mask:   255.255.255.0
               11111111.11111111.11111111.00000000
```

In this example, the first 24 bits (three octets of all 1s in the mask) identify the **network** as `192.168.1`, and the last 8 bits (one octet of all 0s in the mask) identify the **host** as `100` within that network.

### How a Device Uses the Subnet Mask

To find the network address of any IP, the device performs a bitwise **AND** operation between the IP address and the subnet mask. AND means: the result is 1 only when both inputs are 1, otherwise the result is 0.

```
IP:       11000000.10101000.00000001.01100100   (192.168.1.100)
Mask: AND 11111111.11111111.11111111.00000000   (255.255.255.0)
      ─────────────────────────────────────────
Result:   11000000.10101000.00000001.00000000   (192.168.1.0)
```

The result `192.168.1.0` is the network address. This tells the device: "I am on the `192.168.1.0` network." If the destination IP, when ANDed with the same mask, gives the same network address, the destination is on the same local network. Otherwise, the destination is remote.

## 10. CIDR Notation

### What CIDR Is

**CIDR** stands for **Classless Inter-Domain Routing**. It is a method of writing an IP address and its subnet mask together in a compact notation. Instead of writing `192.168.1.100` with a separate subnet mask of `255.255.255.0`, you write:

```
192.168.1.100/24
```

The number after the forward slash is called the **prefix length** (sometimes called the **CIDR prefix** or **prefix**), and it tells you how many bits in the subnet mask are set to 1. In this case, `/24` means the subnet mask has 24 bits set to 1, which corresponds to `255.255.255.0`.

CIDR was introduced in 1993 to replace classful addressing and to allow more flexible allocation of IP address space. With CIDR, you can create subnets of any size, not just the fixed Class A, B, or C sizes.

### Common CIDR Prefixes and Their Subnet Masks

```
CIDR    Subnet Mask         Binary (last octet)    Hosts
/8      255.0.0.0           00000000               16,777,214
/16     255.255.0.0         00000000               65,534
/24     255.255.255.0       00000000               254
/25     255.255.255.128     10000000               126
/26     255.255.255.192     11000000               62
/27     255.255.255.224     11100000               30
/28     255.255.255.240     11110000               14
/29     255.255.255.248     11111000               6
/30     255.255.255.252     11111100               2
/31     255.255.255.254     11111110               0 (point-to-point, special case)
/32     255.255.255.255     11111111               1 (single host)
```

### Hosts Formula

The number of **usable host addresses** in any subnet is calculated as:

```
Usable hosts  =  2^(32 − prefix_length) − 2
```

You subtract 2 because the first address is the network address and the last address is the broadcast address, and neither can be assigned to a host.

**Examples:**

- `/24`: 2^(32−24) − 2 = 2⁸ − 2 = 256 − 2 = **254 usable hosts**
- `/25`: 2^(32−25) − 2 = 2⁷ − 2 = 128 − 2 = **126 usable hosts**
- `/26`: 2^(32−26) − 2 = 2⁶ − 2 = 64 − 2 = **62 usable hosts**
- `/27`: 2^(32−27) − 2 = 2⁵ − 2 = 32 − 2 = **30 usable hosts**
- `/28`: 2^(32−28) − 2 = 2⁴ − 2 = 16 − 2 = **14 usable hosts**
- `/30`: 2^(32−30) − 2 = 2² − 2 = 4 − 2 = **2 usable hosts** (used for point-to-point links)
- `/32`: 2^(32−32) − 2 = 2⁰ − 2 = 1 − 2 = technically negative, but `/32` is a special case meaning a single specific host address, used in firewall rules and routing to refer to exactly one IP.

## 11. Subnetting — The Most Critical Skill

Subnetting is the process of dividing a larger network into smaller sub-networks called **subnets**. Organizations use subnetting to organize their networks logically (separating departments, separating servers from workstations, isolating sensitive systems), to improve performance, and to enhance security by controlling traffic flow between segments.

For an ethical hacker, once you know the network range of a target's subnet, you know exactly which IP addresses exist within it and exactly which ones to scan.

### The Block Size Concept

The **block size** for a subnet tells you how many IP addresses fit in each subnet. For subnetting in the last octet, the block size is calculated as:

```
Block size  =  256 − value of last octet of the subnet mask
```

For example, with a mask of `255.255.255.192` (which is `/26`), the last octet is 192:

```
Block size  =  256 − 192  =  64
```

This means subnets in this range start at intervals of 64 in the last octet: `0, 64, 128, 192`.

### Step-by-Step Subnet Calculation

Given any IP address and CIDR prefix, you can calculate all the key information about that subnet.

**Example: Calculate the subnet details for `192.168.10.50/26`**

**Step 1 — Find the subnet mask.**

The prefix is `/26`, meaning 26 bits are 1. Write them out:

```
11111111.11111111.11111111.11000000
```

Convert the last octet `11000000` to decimal: 128 + 64 = **192**

Subnet mask = `255.255.255.192`

**Step 2 — Find the block size.**

```
Block size  =  256 − 192  =  64
```

Subnets start at: 0, 64, 128, 192 in the last octet.

**Step 3 — Find which subnet block the IP falls in.**

The IP is `192.168.10.50`. The last octet is `50`. Look at the subnet boundaries: `0, 64, 128, 192`. The value `50` falls between `0` and `64`, so it belongs to the subnet starting at `0`.

**Step 4 — Calculate network address, broadcast, first host, last host.**

```
Network address:    192.168.10.0     (first address of the block)
Broadcast address:  192.168.10.63    (last address of the block: 0 + 64 − 1 = 63)
First usable host:  192.168.10.1     (network address + 1)
Last usable host:   192.168.10.62    (broadcast address − 1)
Total usable hosts: 62               (2⁶ − 2)
```

### Another Example: `172.16.45.200/20`

**Step 1 — Subnet mask:**

20 bits are 1. The first two octets are fully 1s (16 bits). The third octet has 4 more bits:

```
Third octet: 11110000  =  240
```

Subnet mask = `255.255.240.0`

**Step 2 — Block size (working in the third octet now because that is where the boundary falls):**

```
Block size  =  256 − 240  =  16
```

Subnets in the third octet start at: 0, 16, 32, 48, 64, ...

**Step 3 — Find the subnet:**

The IP is `172.16.45.200`. The third octet is `45`. It falls between `32` and `48`, so the subnet starts at `32` in the third octet.

**Step 4 — Calculate all values:**

```
Network address:    172.16.32.0
Broadcast address:  172.16.47.255    (32 + 16 − 1 = 47 in third octet, last octet = 255)
First usable host:  172.16.32.1
Last usable host:   172.16.47.254
Total usable hosts: 4094             (2¹² − 2)
```

### Why Subnetting Matters in Ethical Hacking

When you are given scope for a penetration test as `192.168.1.0/24`, you immediately know you have 254 potential host addresses to scan (`192.168.1.1` through `192.168.1.254`). When you see a corporate network running `10.0.0.0/8`, you know there are over 16 million potential addresses. When you enumerate a target's network and discover they have a `/30` subnet between two routers, you know it is a point-to-point link. This contextual awareness comes entirely from understanding subnetting.

## 12. NAT — Network Address Translation

### What NAT Is

**NAT** is the technology that allows multiple devices on a private network to share a single public IP address when communicating with the internet. NAT is typically performed by a router or firewall at the border between the private network and the internet.

### How NAT Works (Step by Step)

1. Your laptop has the private IP address `192.168.1.10` and wants to send a request to the web server at `93.184.216.34` (example.com).
2. The laptop sends the packet to its default gateway (the router), with source IP `192.168.1.10` and destination IP `93.184.216.34`.
3. The router replaces the source IP `192.168.1.10` with its own public IP address (for example, `203.0.113.5`) before forwarding the packet out to the internet. It records this translation in a **NAT table**.
4. The web server at `93.184.216.34` receives the request, sees the source as `203.0.113.5`, and sends its response back to `203.0.113.5`.
5. The router receives the response, looks up `203.0.113.5` in its NAT table, determines that this response belongs to `192.168.1.10`, rewrites the destination IP back to `192.168.1.10`, and forwards it to your laptop.

Your laptop never knows any of this happened. From its perspective, it sent a request and got a response.

### PAT — Port Address Translation

The most common form of NAT in home and office networks is actually **PAT** (Port Address Translation), also called **NAT Overload**. Because many devices share a single public IP, the router uses **port numbers** to distinguish between the different conversations. For example:

```
192.168.1.10:5421  →  203.0.113.5:10001   (laptop)
192.168.1.11:7234  →  203.0.113.5:10002   (desktop)
192.168.1.12:9124  →  203.0.113.5:10003   (phone)
```

All three devices are sharing `203.0.113.5`, but the router knows which response belongs to which device based on the port number.

### Ethical Hacking Implications of NAT

From the outside (internet perspective), you see only one or a few public IP addresses for an organization even though hundreds or thousands of devices may be behind them. This is why external reconnaissance gives you a limited view. When you are doing an internal penetration test and you are inside the network, you suddenly see the full internal IP space with all its individual devices — a very different and much richer picture.

NAT also means that you cannot directly initiate a connection from the internet to a device behind NAT unless the router has been explicitly configured to **port-forward** that traffic.

## 13. DHCP vs. Static IP Addresses

### DHCP — Dynamic Host Configuration Protocol

**DHCP** is a network service (running on a server or router) that automatically assigns IP configuration to devices when they connect to a network. When a device joins a network, it broadcasts a request asking "Is there a DHCP server here? I need an IP address." The DHCP server responds by assigning the device an IP address, subnet mask, default gateway, and DNS server addresses — everything the device needs to communicate.

The IP address assigned by DHCP is called a **lease**, and it is temporary. After the lease expires (which could be hours or days), the device must renew it, and it might receive the same or a different IP address. This means that workstations and user devices in most networks do not have a fixed, permanent IP address.

### Static IP Addresses

A **static IP address** is one that is manually configured on a device and never changes. Any device that needs to be reliably reachable at the same address always uses a static IP. Servers, routers, switches, firewalls, printers, and any infrastructure device use static IPs.

### Ethical Hacking Perspective

Static IPs are your primary targets during a penetration test. Servers, firewalls, routers, and network appliances all have static IPs because they need to be reachable consistently. Workstations typically use DHCP, and while they are still targets in an internal test, knowing that they are dynamically addressed helps you focus on the more predictable infrastructure. When you scan a network and find something listening on standard service ports (80, 443, 22, 3389), it almost certainly has a static IP.

## 14. How IP Packets Travel — Routing Basics

### What Routing Is

**Routing** is the process by which routers forward IP packets from one network to another until they reach their destination. Understanding routing at a basic level helps you understand network topology, which is essential for planning attacks and understanding how traffic flows.

### How a Router Decides Where to Forward a Packet

Every router maintains a **routing table** — a list of known networks and the interface or next-hop address to use to reach each one. When a packet arrives, the router looks at the destination IP address and finds the most specific matching entry in its routing table (the entry with the longest prefix length). It then forwards the packet out of the corresponding interface toward the next hop.

### The Default Route

The **default route**, written as `0.0.0.0/0` in IPv4, is the catch-all route that matches any destination not matched by a more specific entry. For a home router, the default route points out toward the ISP. For an end device (your laptop), the default route points to the default gateway. The default route is sometimes called the **"route of last resort."**

### TTL — Time To Live

Every IPv4 packet contains a field called **TTL** (Time To Live), which is a counter set by the sender (commonly to 64 or 128). Every router that forwards the packet decrements the TTL by 1. If the TTL reaches 0, the router drops the packet and sends back an **ICMP Time Exceeded** message to the sender. TTL exists to prevent packets from looping forever in misconfigured networks.

The `traceroute` (Linux/Mac) and `tracert` (Windows) commands exploit this TTL mechanism to map the path packets take across a network — they send packets with progressively increasing TTL values (1, 2, 3...) and collect the ICMP Time Exceeded messages from each router along the path.

## 15. Practical Commands — Finding IP Information

These are the commands you will use on a daily basis as an ethical hacker and security professional.

### On Windows

```cmd
ipconfig
```

Shows your IP address, subnet mask, and default gateway for each network interface.

```cmd
ipconfig /all
```

Shows all the above plus MAC addresses, DHCP status, DNS server addresses, and lease information.

```cmd
ipconfig /release
```

Releases (gives up) the DHCP-assigned IP address.

```cmd
ipconfig /renew
```

Requests a new IP address from the DHCP server.

```cmd
nslookup google.com
```

Queries DNS to find the IP address of a domain name. One of the most useful reconnaissance commands.

```cmd
tracert google.com
```

Traces the route that packets take from your machine to the target, showing each router (hop) along the way. Useful for network mapping.

```cmd
arp -a
```

Displays the ARP cache — the local table mapping IP addresses to MAC addresses. This shows you every device your machine has recently communicated with on the local network.

```cmd
netstat -an
```

Shows all current network connections and listening ports with their IP addresses and port numbers.

### On Linux / macOS

```bash
ip a
```

Shows all network interfaces and their IP addresses (modern Linux). This is the preferred modern command.

```bash
ifconfig
```

Older command for showing network interface information. Still available on macOS and older Linux systems.

```bash
ip route
```

Shows the routing table, including the default gateway.

```bash
ip route show default
```

Shows only the default route, which tells you the default gateway IP address.

```bash
dig google.com
```

Performs a detailed DNS query. Gives much more information than `nslookup`, including the DNS record type, TTL, and which DNS server responded.

```bash
host google.com
```

Simple and fast DNS lookup. Shows the IP address associated with a domain name.

```bash
nslookup google.com
```

Works on Linux/Mac as well. Queries the default DNS server for a domain.

```bash
traceroute google.com
```

Traces the route to the destination, showing each hop. Same purpose as `tracert` on Windows.

```bash
arp -a
```

Displays the ARP cache showing IP-to-MAC address mappings for local network devices.

```bash
netstat -an
```

Shows all active connections and listening ports.

```bash
ss -an
```

Modern Linux replacement for `netstat`. Faster and more feature-rich.

### Finding Your Public IP (Command Line)

```bash
curl ifconfig.me
```

Makes an HTTP request to `ifconfig.me`, which returns your public IP address as seen from the internet. Essential for checking what IP address you appear to be coming from when doing external testing.

```bash
curl ipinfo.io
```

Returns your public IP along with geolocation and ISP information in JSON format.

### Nmap — IP Range Scanning

```bash
nmap 192.168.1.0/24
```

Scans all 256 IP addresses in the subnet for live hosts and open ports. This is the most fundamental active reconnaissance command in ethical hacking.

```bash
nmap -sn 192.168.1.0/24
```

Ping scan only — discovers which hosts are alive in the range without scanning ports. Faster and less noisy.

```bash
nmap -p 1-65535 192.168.1.100
```

Scans all 65,535 TCP ports on a single target.

## 16. IP Geolocation and WHOIS

### WHOIS

Every public IP address block is registered to an organization in a regional internet registry. **WHOIS** is the protocol (and command) for querying these registries to find out who owns an IP address or domain name.

```bash
whois 8.8.8.8
```

This will tell you that `8.8.8.8` is owned by Google LLC, show the registered address of the organization, and provide contact information (abuse contacts, etc.). WHOIS lookup is a standard OSINT technique for identifying a target organization's IP space.

Regional registries and their coverage:

|Registry|Region|
|---|---|
|ARIN|North America|
|RIPE NCC|Europe, Middle East, Central Asia|
|APNIC|Asia-Pacific|
|LACNIC|Latin America and Caribbean|
|AFRINIC|Africa|

### IP Geolocation

**Geolocation** is the process of determining the approximate physical location of an IP address. Every public IP is registered to an organization, and that registration includes a country and often a city. Geolocation databases use this information to estimate where a device using a particular IP is physically located.

Important caveat: **Geolocation is approximate**. It tells you where the IP address is registered or where the ISP infrastructure is, not necessarily where the actual device is. A VPN user in India may appear to be in the Netherlands. A CDN server may show a location far from the actual end user.

Useful tools and resources for geolocation and IP intelligence:

```
whois          — command-line WHOIS lookup
ipinfo.io      — web and API-based IP geolocation and ASN lookup
shodan.io      — internet-wide scanner showing open services on any IP
arin.net       — ARIN's web interface for IP registration lookups
bgp.he.net     — BGP and ASN information, IP block ownership
```

### ASN — Autonomous System Number

An **ASN** (Autonomous System Number) identifies a network under a single administrative domain that presents a consistent routing policy to the internet. Large organizations, ISPs, and cloud providers have their own ASNs. When doing reconnaissance, finding an organization's ASN lets you identify all the IP blocks they own, which may be many different ranges. Tools like `bgp.he.net` and `ipinfo.io` can map an organization to its ASN and show all associated IP ranges — this is more comprehensive than searching for individual IP addresses.

## 17. IP Addresses in the Context of Attacks and Defense

### IP Spoofing

**IP spoofing** is the act of forging the source IP address in an IP packet to make it appear to come from a different address than it actually does. An attacker might do this to impersonate a trusted machine, to hide their true identity, or to reflect attack traffic off a third party (as in DDoS amplification attacks where the attacker spoofs the victim's IP and sends requests to servers that respond to the victim).

However, IP spoofing has an important limitation: because of how TCP works (requiring a three-way handshake where both sides must send and receive sequenced packets), you cannot spoof a TCP connection unless you can intercept the responses. Spoofing works well for UDP-based attacks and for sending single packets, but not for full interactive TCP sessions (unless you are on the same network segment).

### Port Scanning

**Port scanning** is the technique of probing IP addresses and ports to discover which hosts are alive and which services are running. The most widely used tool is **Nmap**. When you perform a port scan, you are sending probes to specific IP addresses and analyzing the responses.

```bash
nmap -sS 192.168.1.0/24    # TCP SYN scan (stealth scan) of entire subnet
nmap -sV 192.168.1.1       # Service version detection on a single host
nmap -O 192.168.1.1        # OS fingerprinting
nmap -A 192.168.1.1        # Aggressive scan: OS, version, scripts, traceroute
```

### Firewall Rules and ACLs

Firewalls and **ACLs** (Access Control Lists) on routers and switches often allow or block traffic based on source or destination IP addresses and port numbers. As a penetration tester, you will frequently encounter IP-based restrictions. Understanding these is essential for finding bypasses (such as approaching from a trusted IP range or tunneling traffic to bypass filters).

### VPN and Tor — Hiding IP Identity

Your public IP address is your identity on the internet. When performing authorized testing, ethical hackers may use **VPNs** (Virtual Private Networks) to route their traffic through a different IP address or country, or use **Tor** (The Onion Router) to route traffic through multiple encrypted hops and exit at a random node worldwide. This is part of **OPSEC** (Operational Security). Note: for authorized penetration tests, always ensure your VPN or proxy usage is within the agreed scope and does not route traffic through unauthorized systems.

### Blacklisting and Whitelisting

IP-based **blacklisting** (blocking specific IPs or ranges) and **whitelisting** (only allowing specific IPs) are common defense mechanisms. Attackers may have their IP blacklisted by threat intelligence feeds if they have previously been associated with malicious activity. As a penetration tester, your testing IP may get blacklisted by a WAF (Web Application Firewall) or IDS (Intrusion Detection System) if you scan too aggressively — something to be aware of during engagements.

### Scanning with IPv6

As mentioned earlier, many targets have both IPv4 and IPv6 connectivity. A common mistake is to scan only IPv4 and miss services exposed on IPv6. Always check for IPv6 addresses during reconnaissance and include IPv6 scanning in your methodology.

```bash
nmap -6 2001:db8::1           # Scan an IPv6 host
```

## Final Notes — What to Master First

If you are studying for certifications like **CompTIA Security+**, **CEH**, or **OSCP**, or preparing for real-world penetration testing, prioritize these skills in order:

**First**, master binary-decimal conversion for octets. You must be able to convert numbers like 192, 168, 224, 240, 252, 255 instantly without a calculator.

**Second**, master subnetting. Given any IP and CIDR prefix, you must be able to instantly identify the network address, broadcast address, first host, last host, and number of usable hosts. Practice until you can do it in your head in under 30 seconds.

**Third**, understand private vs. public addressing and know the RFC 1918 ranges cold. You will reference them in every single internal penetration test.

**Fourth**, be comfortable with the practical commands: `ip a`, `ip route`, `nmap`, `whois`, `dig`, and `traceroute`. These are your daily tools.

**Fifth**, understand NAT and how it affects what you can see from inside vs. outside a network. This shapes your entire methodology depending on whether you are doing an internal or external engagement.

Everything in networking security — and therefore everything in ethical hacking — ultimately comes back to IP addresses. Time invested mastering this topic compounds across every other topic you study after it.
