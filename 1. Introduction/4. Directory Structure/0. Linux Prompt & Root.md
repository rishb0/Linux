# Linux Prompt & "Root" — Explained

---
## Part 1: The Linux Shell Prompt

The prompt is the text displayed in the terminal waiting for your input. Its format varies by shell and distro, but every part carries meaning.

---
### Kali Linux Prompt (Zsh)

```
┌──(rishi㉿kali)-[/]
└─$
```

|Part|Meaning|
|---|---|
|`┌──(`|Opening bracket — start of prompt line 1|
|`rishi`|Current **logged-in username**|
|`㉿`|Kali's custom separator symbol (decorative; replaces `@`)|
|`kali`|**Hostname** of the machine|
|`)-[`|Separator before directory|
|`/`|**Current working directory** (`/` = root dir, `~` = home dir)|
|`]`|Closing bracket for directory|
|`└─$`|Second line; `$` = **prompt symbol**|
|`$`|Indicates a **regular user** session|
|`#`|Indicates a **root user** session|

So if you see:

```
┌──(root㉿kali)-[/root]
└─#
```

— you are logged in as **root**.

---
### Standard Bash Prompt

Most other distros (Ubuntu, CentOS, Debian, etc.) use a single-line Bash prompt:

```
[rishi@centos ~]$
```

| Part     | Meaning                                             |
| -------- | --------------------------------------------------- |
| `[`      | Opening bracket                                     |
| `rishi`  | Current **username**                                |
| `@`      | Separator — means "at"                              |
| `centos` | **Hostname**                                        |
| `~`      | **Current directory** (`~` = user's home directory) |
| `]`      | Closing bracket                                     |
| `$`      | Regular user                                        |
| `#`      | Root user                                           |


---
### Ubuntu / Debian Default Bash Prompt

```
rishi@ubuntu:~$
```

|Part|Meaning|
|---|---|
|`rishi`|Username|
|`@`|"at" separator|
|`ubuntu`|Hostname|
|`:`|Separator|
|`~`|Current directory|
|`$`|Regular user (`#` for root)|

---
### Prompt Symbol Summary

| Symbol | Meaning                             |
| ------ | ----------------------------------- |
| `$`    | Regular / non-root user             |
| `#`    | Root user (full privileges)         |
| `%`    | Zsh default prompt for regular user |

> **Rule of thumb:** Whenever you see `#` at the end of a prompt — you are root.

---
### Common Directory Shortcuts in the Prompt

| Symbol | Expands To                                                           |
| ------ | -------------------------------------------------------------------- |
| `~`    | Current user's home directory (`/home/username` or `/root` for root) |
| `/`    | Root directory (top of the filesystem)                               |
| `.`    | Current directory                                                    |
| `..`   | Parent directory                                                     |
| `-`    | previous working directory                                           |

---
### The `PS1` Variable — What Controls the Prompt

The prompt appearance is controlled by the shell variable `PS1`.

View your current prompt definition:

```bash
echo $PS1
```

Example output on Ubuntu:

```
\u@\h:\w\$
```

|Code|Meaning|
|---|---|
|`\u`|Username|
|`\h`|Hostname (short)|
|`\H`|Hostname (full)|
|`\w`|Current working directory (full path)|
|`\W`|Current working directory (basename only)|
|`\$`|`$` for regular user, `#` for root|
|`\d`|Current date|
|`\t`|Current time (HH:MM:SS)|

Temporarily change your prompt:

```bash
PS1="[\u@\h \W]\$ "
```

## Part 2: Everything Called "Root" in Linux

The word **"root"** is one of the most overloaded terms in Linux. It refers to several completely different things depending on context.

---
### 1. Root User (`root`)

- The **superuser** account on every Linux system.
- **UID = 0** — the kernel grants unrestricted access to any process with UID 0.
- Can read, write, and execute any file regardless of permissions.
- Can kill any process, modify any system file, and load/unload kernel modules.
- Home directory: `/root`
- Prompt symbol: `#`

```bash
whoami        # shows: root
id            # shows: uid=0(root) gid=0(root) groups=0(root)
```

Switch to root from a regular user:

```bash
sudo su -
```

or directly (if root password is set):

```bash
su -
```

> **Security note:** Logging in directly as root is discouraged in production. Use `sudo` for individual privileged commands instead.

---
### 2. Root Directory (`/`)

- The **top-level directory** of the entire Linux filesystem.
- Everything — all files, all directories, all mounted drives — lives under `/`.
- There is no directory above `/`. It is the absolute beginning of all paths.
- Not to be confused with `/root` (which is the root _user's_ home directory).

```
/              ← root directory
├── etc/
├── home/
├── root/      ← root user's home (different thing)
├── var/
└── ...
```

- An **absolute path** always starts with `/` (e.g., `/etc/passwd`).
- A **relative path** does not start with `/` (e.g., `etc/passwd` — relative to current dir).

---
### 3. Root's Home Directory (`/root`)

- The **home directory of the root user**.
- Equivalent to `/home/username` for regular users — but root gets `/root` instead.
- Contains root's personal files, SSH keys, shell history, and scripts.
- Not accessible by regular users (permissions: `drwx------`).

```bash
ls /root          # permission denied for regular users
sudo ls /root     # accessible with sudo
```

🔴 **Pentesting:** Gaining access to `/root` = full system compromise. Contains `.bash_history`, `.ssh/`, and often sensitive scripts.

---
### 4. Root Filesystem

- The **filesystem mounted at `/`** — the primary filesystem the OS boots from.
- All other filesystems (other partitions, USB drives, network shares) are **mounted** as subdirectories within the root filesystem.
- Defined in `/etc/fstab`.

```bash
df -h /           # shows disk usage of the root filesystem
mount | grep "on / "   # shows what device is mounted as root fs
```

---
### 5. Root Cause (in `/lost+found`)

- Each filesystem partition has a `/lost+found` directory.
- Used by `fsck` (filesystem check) to store **recovered file fragments** after a crash or corruption.
- Not "root" by name, but tied to the root of each filesystem partition.

---
### 6. Sudo / Root Privileges

- **`sudo`** (superuser do) allows a regular user to run specific commands **as root** without switching to the root account.
- Controlled by `/etc/sudoers`.
- Safer than logging in as root directly — provides an audit trail.

Run a single command as root:

```bash
sudo command
```

Open a root shell via sudo:

```bash
sudo -i       # login shell as root
sudo -s       # non-login shell as root
```

Check what sudo privileges your user has:

```bash
sudo -l
```

---
### 7. Root in Context of Web Servers (`webroot`)

- The **document root** — the directory a web server serves files from.
- Default locations:
    - Apache / Nginx on Debian/Ubuntu: `/var/www/html`
    - Apache on RHEL/CentOS: `/var/www/html`
    - Nginx custom configs: defined in `root` directive in `/etc/nginx/sites-available/`

Not the same as the Linux root user or root directory — just named "root" in web server config terminology.

---
## Quick Reference — All "Roots" at a Glance

| Term                  | What It Is                                   | Path / Value               |
| --------------------- | -------------------------------------------- | -------------------------- |
| **Root user**         | Superuser account; UID 0; full system access | username: `root`, UID: `0` |
| **Root directory**    | Top of the entire filesystem                 | `/`                        |
| **Root's home dir**   | Root user's personal home folder             | `/root`                    |
| **Root filesystem**   | The partition mounted at `/`                 | mounted at `/`             |
| **Webroot**           | Web server document directory                | `/var/www/html` (default)  |
| **`/lost+found`**     | Recovered file fragments per partition       | `/<mountpoint>/lost+found` |
| **sudo / root privs** | Mechanism to run commands as root            | via `/etc/sudoers`         |
