# Windows Directory Structure

The Windows directory structure is a **hierarchical system** that organizes files and folders starting from a **root directory**, branching down into subdirectories. Every drive has its own root.

---
## 1. Root Directory (`C:\`)

- The **top-level directory** of a drive вЂ” all files and folders branch from here.
- The main system drive is typically `C:\`.
- Additional drives (`D:\`, `E:\`, etc.) have their own root directories.

Common top-level folders under `C:\`:

```
C:\Windows
C:\Program Files
C:\Program Files (x86)
C:\Users
C:\ProgramData
C:\Temp
```

---
## 2. System Folders

### `C:\Windows`

Core operating system files.

| Subfolder  | Purpose                                                   |
| ---------- | --------------------------------------------------------- |
| `System32` | Critical system executables and DLL libraries             |
| `WinSxS`   | Multiple versions of system libraries (for compatibility) |
| `Temp`     | Temporary files used by the OS                            |
| `Fonts`    | All installed system fonts                                |
| `Logs`     | System logs for debugging and diagnostics                 |

---
### `C:\Program Files` and `C:\Program Files (x86)`

| Folder                   | Purpose                                             |
| ------------------------ | --------------------------------------------------- |
| `C:\Program Files`       | Installs **64-bit** applications                    |
| `C:\Program Files (x86)` | Installs **32-bit** applications on a 64-bit system |

Each installed application gets its own subfolder.

- Example: `C:\Program Files\Google\Chrome`

---

### `C:\Users`

Stores user-specific data and settings for every account on the machine.

|Subfolder|Purpose|
|---|---|
|`Administrator`|Default admin account folder|
|`Public`|Shared files accessible to all users|
|`<Username>`|Personalized folder for each individual user|

Each user's folder (`C:\Users\<Username>\`) contains:

|Subfolder|Purpose|
|---|---|
|`Desktop`|Files and shortcuts visible on the user's desktop|
|`Documents`|Personal documents|
|`Downloads`|Files downloaded from the internet|
|`AppData`|**Hidden** folder for user-specific application data|

`AppData` is split into three subfolders:

|Subfolder|Purpose|
|---|---|
|`Local`|Non-roaming data; specific to this machine only|
|`Roaming`|Data that syncs across devices using the same account|
|`LocalLow`|Low-integrity apps (e.g., browsers in protected mode)|

---
### `C:\ProgramData`

- Stores **shared application data** accessible to all users on the machine.
- Hidden by default.
- Example: `C:\ProgramData\Microsoft\Windows\Start Menu\Programs`
- `C:\Users\All Users` is a legacy symlink pointing here вЂ” same location.

---

### `C:\Temp` _(Optional)_

- Temporary storage for applications and system processes.
- Not always present; some systems use `C:\Windows\Temp` instead.

---

## 3. Additional System Folders

|Folder|Purpose|
|---|---|
|`C:\$Recycle.Bin`|Holds deleted files before permanent removal|
|`C:\System Volume Information`|System restore points and VSS recovery data (protected)|
|`C:\Boot`|Bootloader files required to start Windows (on bootable drives)|

---

## 4. Logical Drives and Partitions

Additional drive letters (`D:\`, `E:\`, etc.) can represent:

- A **separate physical drive** (second HDD/SSD)
- A **partition** on the same physical drive
- A **removable device** (USB drive, CD/DVD)

Each has its own independent root directory and folder structure.

## 6. Command-Line Utilities

List files and folders in a directory:

```cmd
dir C:\Windows
```

Display the full directory tree:

```cmd
tree C:\
```

Display tree including files (not just folders):

```cmd
tree /F
```

Navigate to a directory:

```cmd
cd C:\Users
```
