# Linux Command Syntax — General Format

---
## Basic Structure

```
command [-options] [argument]
```

|Part|Meaning|
|---|---|
|`command`|The program or utility to run (e.g., `ls`, `cp`, `ssh`)|
|`[-options]`|Optional flags that modify the command's behaviour|
|`[argument]`|The target the command acts on (file, path, user, IP, etc.)|

Square brackets `[ ]` mean the part is **optional** — not all commands need options or arguments.

---
## Options / Switches

Options come in two forms:

### Short form — single dash + single letter

```bash
ls -a
```
### Long form — double dash + word

```bash
ls --all
```

Short and long form of the same option are usually equivalent:

```bash
ls -a
```

```bash
ls --all
```

### Combining short options — stack letters after one dash

```bash
ls -la
ls -l -a        # same thing
```

### Passing a value to an option

```bash
ssh -p 2222 user@ip       # short form with value
ssh --port 2222 user@ip   # long form with value (if supported)
```

---
## Arguments

The target the command operates on — a file, directory, hostname, username, etc.

Single argument:

```bash
cat /etc/passwd
```

Multiple arguments:

```bash
cp file1.txt file2.txt /destination/
```

---
## Getting Help

Show built-in help for any command:

```bash
command --help
```

Open the full manual page:

```bash
man command
```

Search for a command by keyword (if you don't know the exact name):

```bash
man -k keyword
```

Show a one-line description of a command:

```bash
whatis command
```

Show the full path of a command's binary:

```bash
which command
```

Show all locations of a command (binary, man page, source):

```bash
whereis command
```

## Special Symbols in Syntax Notation

When reading documentation or man pages, these symbols have standard meanings:

|Symbol|Meaning|
|---|---|
|`[ ]`|Optional — can be omitted|
|`< >`|Required — must be provided|
|`\|`|OR — choose one of the options|
|`...`|Repeatable — can provide multiple values|
|`{ }`|Choose exactly one from the listed group|

Example from a man page:

```
cp [OPTION]... SOURCE DEST
```

Means: options are optional, but SOURCE and DEST are required, and options can be repeated.

|`-a`|show hidden files|Short option|
|`/etc`|`/etc`|Argument (target directory)|

## Common Patterns

Run a command as superuser:

```bash
sudo command
```

Redirect output to a file (overwrite):

```bash
command > output.txt
```

Redirect output to a file (append):

```bash
command >> output.txt
```

Pipe output of one command into another:

```bash
command1 | command2
```

Suppress all output (send to /dev/null):

```bash
command > /dev/null 2>&1
```

Run command in the background:

```bash
command &
```

Chain commands — run next only if previous succeeded:

```bash
command1 && command2
```

Chain commands — run next only if previous failed:

```bash
command1 || command2
```

Run multiple commands regardless of outcome:

```bash
command1 ; command2
```
