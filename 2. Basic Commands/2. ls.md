# `ls` — List Directory Contents

Lists files and directories inside a given directory. Without any argument, lists the current directory.

- `ls` → Standard command to list directory contents  
- `/bin/ls` → Absolute path (bypass alias)  
- `command ls` → Ignore alias/function  
- `\ls` → Escape alias  

- `echo *` → Lists files (no hidden, simple glob)  
- `printf "%s\n" *` → Safer glob listing  

- `find . -maxdepth 1` → List files in current dir  
- `find . -maxdepth 1 -type f` → Only files  
- `find . -maxdepth 1 -type d` → Only directories  

- `stat *` → Shows file info (acts like listing)  
- `du -a` → Lists files with sizes  

- `tree` → Recursive directory listing (if installed)  

- `python3 -c "import os; print(os.listdir())"` → Python method  
- `perl -e 'opendir(D,\".\"); print join(\"\n\", readdir(D))'` → Perl method  

- `for f in *; do echo "$f"; done` → Bash loop listing  

---

```bash
ls → clean listing  
glob (*) → shell expansion  
find → powerful filtering  
```

---
## Syntax

```
ls [options] [path]
```

---

## Basic Usage

List current directory:

```bash
ls
```

List a specific directory:

```bash
ls /etc
```

List multiple paths at once:

```bash
ls /etc /var /home
```

---

## Most Used Options

### `-l` — Long listing format

Shows permissions, owner, group, size, date, and filename.

```bash
ls -l
```

Output format:

```
-rwxr-xr-x  1  root  root  4096  Jan 10 12:00  filename
│            │  │     │     │     │             │
│            │  │     │     │     │             └─ filename
│            │  │     │     │     └─ last modified date
│            │  │     │     └─ size in bytes
│            │  │     └─ group
│            │  └─ owner
│            └─ hard link count
└─ permissions (filetype + rwx for user/group/others)
```

---
### `-a` — Show all files including hidden

Hidden files start with `.` (e.g., `.bashrc`, `.ssh`).

```bash
ls -a
```

### `-A` — Show hidden files but exclude `.` and `..`

```bash
ls -A
```

### `-h` — Human-readable file sizes

Used with `-l`; shows sizes as KB, MB, GB instead of raw bytes.

```bash
ls -lh
```

### `-r` — Reverse order

```bash
ls -r
```

### `-t` — Sort by modification time (newest first)

```bash
ls -t
```

### `-S` — Sort by file size (largest first)

```bash
ls -S
```

### `-R` — Recursive — list all subdirectories too

```bash
ls -R
```

### `-i` — Show inode number of each file

```bash
ls -i
```

### `-d` — List directory itself, not its contents

```bash
ls -d /etc
```

### `--color` — Coloured output (usually on by default)

```bash
ls --color
```

### `-1` — One file per line

```bash
ls -1
```

### `-n` — Like `-l` but shows UID and GID numbers instead of names

```bash
ls -n
```

## Common Combinations

Long list with hidden files:

```bash
ls -la
```

Long list, human-readable sizes, hidden files:

```bash
ls -lah
```

Long list sorted by modification time (newest first):

```bash
ls -lt
```

Long list sorted by size (largest first):

```bash
ls -lS
```

Long list sorted by time, reversed (oldest first):

```bash
ls -ltr
```

Long list of a specific directory:

```bash
ls -l /var/log
```

List only directories in current path:

```bash
ls -d */
```

## Understanding the Permission String

The first column in `ls -l` output is the permission string — 10 characters:

```
- r w x r w x r - x
│ │     │     │
│ │     │     └─ others (world): r-x = read + execute
│ │     └─ group: rwx = read + write + execute
│ └─ user/owner: rwx = read + write + execute
└─ file type:
     -  = regular file
     d  = directory
     l  = symbolic link
     c  = character device
     b  = block device
     p  = named pipe
     s  = socket
```
