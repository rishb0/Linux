# `pwd` — Print Working Directory

Prints the full absolute path of the directory you are currently in.

- `pwd`
- `getcwd`
- `echo $PWD`

---
## Syntax

```bash
pwd [option]
```

---

## Options

|Option|Meaning|
|---|---|
|`pwd`|Default — prints the logical path (follows symlinks as-is)|
|`pwd -L`|**Logical** — same as default; shows the path with symlinks unresolved|
|`pwd -P`|**Physical** — resolves all symlinks and shows the real absolute path|

---
## Examples

Print current directory:

```bash
pwd
```

Print logical path (default behaviour — same as no option):

```bash
pwd -L
```

Print physical path (resolve symlinks):

```bash
pwd -P
```

---

## `-L` vs `-P` — When It Matters

If you navigated into a directory via a **symlink**, `-L` and `-P` give different results:

```bash
cd /var/www          # /var/www is a symlink to /srv/webroot
```

```bash
pwd -L               # output: /var/www         (symlink path)
```

```bash
pwd -P               # output: /srv/webroot      (real path)
```

---
## Common Usage

Store current path in a variable:

```bash
DIR=$(pwd)
echo $DIR
```

Use `pwd` inside a script to get the script's working directory:

```bash
CURRENT=$(pwd)
echo "Running from: $CURRENT"
```

---
## Notes

- Output always starts with `/` — it is always an absolute path.
- `pwd` is a **shell built-in** in Bash and Zsh, and also exists as `/usr/bin/pwd` (standalone binary). Both behave the same.
- The current directory is also stored in the shell variable `$PWD` — equivalent to running `pwd -L`.

```bash
echo $PWD       # same output as pwd
```
