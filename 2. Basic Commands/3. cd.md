# `cd` — Change Directory

Navigates between directories in the filesystem.


---
## Syntax

```
cd [path]
```

---

## Basic Usage

Go to a specific directory using an absolute path:

```bash
cd /etc
```

Go to a directory using a relative path (from current location):

```bash
cd Documents
```

Go to home directory (any of these work):

```bash
cd
```

```bash
cd ~
```

```bash
cd $HOME
```

Go to root directory:

```bash
cd /
```

Go up one level (parent directory):

```bash
cd ..
```

Go up two levels:

```bash
cd ../..
```

Go to a sibling directory (up one then into another):

```bash
cd ../otherfolder
```

Go to previous directory (toggle between last two locations):

```bash
cd -
```

---

## Absolute vs Relative Paths

An **absolute path** starts from `/` — works from anywhere:

```bash
cd /var/log
```

A **relative path** starts from current directory — depends on where you are:

```bash
cd log
```

---

## Special Path Symbols

|Symbol|Meaning|
|---|---|
|`/`|Root directory|
|`~`|Current user's home directory|
|`.`|Current directory|
|`..`|Parent directory|
|`-`|Previous directory (last location)|
|`$HOME`|Same as `~` — environment variable for home|

---

## Navigating Paths with Spaces

If a directory name contains spaces, wrap in quotes or escape with `\`:

Using quotes:

```bash
cd "My Folder"
```

Using backslash to escape the space:

```bash
cd My\ Folder
```

---

## Useful Combinations

Go to home then immediately check location:

```bash
cd ~ && pwd
```

Go to a directory and list its contents:

```bash
cd /var/log && ls -la
```

---
## Notes

- `cd` is a **shell built-in** — not an external binary. It exists inside the shell itself (`bash`, `zsh`, etc.) because changing directory must affect the current shell process; an external program cannot change its parent's directory.
- Running `which cd` returns nothing or a path to a wrapper — the real `cd` is always the built-in.
- `cd -` prints the previous path and switches to it — useful for quickly toggling between two working directories.
